# Internationalization (i18n)

This guide explains how to set up multilingual content with vike-content-collection. Two strategies are supported: **slug suffix** (default) and **metadata field**. Choose the one that fits your project structure.

## Strategy 1: Slug suffix (recommended)

The simplest approach. Each localized version of a page is a separate file with the locale appended to the filename.

### Folder structure

```
pages/
└── docs/
    ├── +Content.ts
    ├── getting-started.md        ← default locale (e.g. English)
    ├── getting-started.fr.md     ← French
    ├── getting-started.de.md     ← German
    ├── installation.md
    ├── installation.fr.md
    └── installation.de.md
```

The base slug is the filename without the locale suffix. `getting-started` is the base, `getting-started.fr` is the French version.

### Define the collection

No special schema changes are needed. The i18n helpers work on top of any existing collection:

```ts
// pages/docs/+Content.ts
import { z } from 'zod'

export const Content = z.object({
  title: z.string(),
  description: z.string().optional(),
})
```

### Content files

Each file has its own frontmatter in the target language:

```md
<!-- pages/docs/getting-started.md -->
---
title: "Getting Started"
description: "Learn how to set up the project"
---

Welcome to the documentation.
```

```md
<!-- pages/docs/getting-started.fr.md -->
---
title: "Commencer"
description: "Apprenez à configurer le projet"
---

Bienvenue dans la documentation.
```

```md
<!-- pages/docs/getting-started.de.md -->
---
title: "Erste Schritte"
description: "Erfahren Sie, wie Sie das Projekt einrichten"
---

Willkommen in der Dokumentation.
```

### Query available locales

```ts
import { getAvailableLocales } from 'vike-content-collection'

const locales = getAvailableLocales('docs', 'getting-started')
// ['', 'de', 'fr']
//  ^-- empty string = the base slug (default locale)
```

### Get a specific locale

```ts
import { getLocalizedEntry } from 'vike-content-collection'

// Get the French version
const frEntry = getLocalizedEntry('docs', 'getting-started', 'fr')
console.log(frEntry?.metadata.title) // 'Commencer'

// Get the default (English) version
const enEntry = getLocalizedEntry('docs', 'getting-started', '')
console.log(enEntry?.metadata.title) // 'Getting Started'
```

### Build a language switcher

```ts
// pages/docs/@slug/+data.ts
import {
  getAvailableLocales,
  getLocalizedEntry,
} from 'vike-content-collection'
import type { PageContext } from 'vike/types'

export function data(pageContext: PageContext) {
  const slug = pageContext.routeParams.slug

  // Determine which base slug and locale this is
  const parts = slug.split('.')
  const locale = parts.length > 1 ? parts[parts.length - 1] : ''
  const baseSlug = locale ? parts.slice(0, -1).join('.') : slug

  const entry = getLocalizedEntry('docs', baseSlug, locale)
  const locales = getAvailableLocales('docs', baseSlug)

  return {
    entry,
    currentLocale: locale || 'en',
    availableLocales: locales,
    baseSlug,
  }
}
```

```tsx
// pages/docs/@slug/+Page.tsx
import { useData } from 'vike-renderer/useData'

export function Page() {
  const { entry, currentLocale, availableLocales, baseSlug } = useData()

  return (
    <div>
      <nav>
        {availableLocales.map((loc) => {
          const label = loc || 'en'
          const href = loc ? `/docs/${baseSlug}.${loc}` : `/docs/${baseSlug}`
          return (
            <a
              key={label}
              href={href}
              style={{ fontWeight: label === currentLocale ? 'bold' : 'normal' }}
            >
              {label.toUpperCase()}
            </a>
          )
        })}
      </nav>

      <h1>{entry?.metadata.title}</h1>
    </div>
  )
}
```

### Custom separator

If you prefer a different separator (e.g. `getting-started-fr.md` instead of `getting-started.fr.md`):

```ts
const locales = getAvailableLocales('docs', 'getting-started', {
  separator: '-',
})

const frEntry = getLocalizedEntry('docs', 'getting-started', 'fr', {
  separator: '-',
})
```

## Strategy 2: Metadata field

If you prefer to keep the locale in frontmatter rather than the filename. This approach uses a single slug structure with a metadata field to distinguish locales.

### Folder structure

```
pages/
└── docs/
    ├── +Content.ts
    ├── getting-started.md        ← locale: "en" in frontmatter
    ├── getting-started.fr.md     ← locale: "fr" in frontmatter
    └── getting-started.de.md     ← locale: "de" in frontmatter
```

The file naming can follow any convention -- what matters is the metadata field.

### Define the collection

Add a `locale` field to your schema:

```ts
// pages/docs/+Content.ts
import { z } from 'zod'

export const Content = z.object({
  title: z.string(),
  locale: z.string(),
})
```

### Content files

```md
<!-- pages/docs/getting-started.md -->
---
title: "Getting Started"
locale: "en"
---

Welcome to the documentation.
```

```md
<!-- pages/docs/getting-started.fr.md -->
---
title: "Commencer"
locale: "fr"
---

Bienvenue dans la documentation.
```

### Query with metadata strategy

Pass `strategy: "metadata"` to use the metadata field instead of slug suffixes:

```ts
import { getAvailableLocales, getLocalizedEntry } from 'vike-content-collection'

const locales = getAvailableLocales('docs', 'getting-started', {
  strategy: 'metadata',
  field: 'locale',     // this is the default
})
// ['de', 'en', 'fr']

const frEntry = getLocalizedEntry('docs', 'getting-started', 'fr', {
  strategy: 'metadata',
})
console.log(frEntry?.metadata.title) // 'Commencer'
```

### Custom field name

If your locale field is named differently:

```ts
const locales = getAvailableLocales('docs', 'getting-started', {
  strategy: 'metadata',
  field: 'lang',
})
```

## Combining with other features

### Localized breadcrumbs

Use `getBreadcrumbs` with locale-aware labels:

```ts
import { getBreadcrumbs } from 'vike-content-collection'

const labels: Record<string, Record<string, string>> = {
  en: { docs: 'Documentation' },
  fr: { docs: 'Documentation' },
  de: { docs: 'Dokumentation' },
}

const crumbs = getBreadcrumbs('docs', baseSlug, {
  labels: labels[currentLocale],
  basePath: `/${currentLocale}`,
})
```

### Localized rendering

Render the localized entry's content as usual:

```ts
import { getLocalizedEntry, renderEntry } from 'vike-content-collection'

const entry = getLocalizedEntry('docs', 'getting-started', 'fr')
if (entry) {
  const { html, headings } = await renderEntry(entry)
}
```

### URL generation with locale prefix

Use `getEntryUrl` with a locale-prefixed base path:

```ts
import { getEntryUrl } from 'vike-content-collection'

const url = getEntryUrl('docs', 'getting-started', {
  basePath: '/fr',
})
// '/fr/docs/getting-started'
```

## API reference

### `LocaleOptions`

| Option      | Type                       | Default    | Description                              |
| ----------- | -------------------------- | ---------- | ---------------------------------------- |
| `strategy`  | `"suffix" \| "metadata"`   | `"suffix"` | How to detect locales                    |
| `field`     | `string`                   | `"locale"` | Metadata field (metadata strategy only)  |
| `separator` | `string`                   | `"."`      | Separator between slug and locale        |

### `getAvailableLocales(name, baseSlug, options?)`

Returns a sorted array of locale strings. With the suffix strategy, an empty string `""` represents the base slug (default locale).

### `getLocalizedEntry(name, baseSlug, locale, options?)`

Returns the entry matching the locale, or `undefined` if not found. With the suffix strategy, pass `""` for the default locale.

---

## Which strategy should I use?

| | Suffix (default) | Metadata |
|---|---|---|
| **Setup** | No schema changes needed | Requires a `locale` field in schema |
| **File naming** | Convention-based: `slug.locale.md` | Any naming works, locale is in frontmatter |
| **Default locale** | Represented by `""` (empty string) | Explicit value like `"en"` |
| **Best for** | Simple sites, clear file naming | Complex setups, locale from CMS or external source |
